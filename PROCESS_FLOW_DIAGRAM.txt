================================================================================
                            ERROR HANDLING FLOWS
================================================================================

FORM VALIDATION ERRORS:
User Input ──► Form Validation ──► Error Display ──► User Correction

API ERRORS:
Frontend ──► API Call ──► Backend Error ──► Error Response ──► Error Toast/Message

DATABASE ERRORS:
API Call ──► Database Operation ──► DB Error ──► Error Log ──► Error Response

================================================================================
                            DATA FLOW SUMMARY
================================================================================

FORWARD FLOW (Create):
User Form → handleChange → formData → handleSubmit → createOrder API
→ POST /api/orders → saveOrder → MongoDB → Response → Success → Navigate

REVERSE FLOW (Display):
Order Page → DataTableDemo → useEffect → getOrders API → GET /api/orders
→ Order.find({}) → Orders Array → Table Display → User View

================================================================================
                            COMPONENT RELATIONSHIPS
================================================================================

CreateOrder.tsx:
├── handleSubmit() → createOrder() → API → Backend → DB
├── handleChange() → formData (state) → handleSubmit()
└── Success → navigate() → Order.tsx

Order.tsx:
└── DataTableDemo → getOrders() → API → Backend → DB → Display

serviceApi.ts:
├── createOrder() → POST /api/orders → saveOrder()
└── getOrders() → GET /api/orders → Order.find({})

order.rout.ts:
├── POST / → saveOrder() → Order Model → MongoDB
└── GET / → Order.find({}) → Orders Array

db.ts:
├── OrderSchema → Order Model → Database Operations
├── saveOrder() → new Order() → order.save()
└── Order Model → MongoDB Collection 'orders'
